Описание бота:

Ссылка на бот: https://t.me/meteo_news_bot
Бот предназначен для взаимодействие пользователей с парсером новостей погоды с сайта https://www.gismeteo.ru/news/

Описание команд бота:
    /start - приветствие пользователя
    /help - список команд бота
    /about - вывод описания бота
    /news n - показать n-ю новость (n от 1 до 7)
    /news all - показать все новости

Описание функционала:
Начало работы определяется запуском файла telegram_bot.py

При запуске создается экземпляр класса News, который представляет собой
непосредственно парсер https страницы https://www.gismeteo.ru/news/

Основные команды бота /news n - для вывода новости по ее номеру и /news all - 
для вывода всех новостей (от 1 до 7).
Обе команды обращаются к методу get_news экземпляра класса News с передачей
в качестве параметра индекса новости (индекс на 1 меньше номера).
Для параметра all вызов функции производится в цикле по всем индексам.

Получив запрос на новость, метод get_news запрашивает обновление списка новостей
Класс News использует 2 уровнz кэширования: в оперативной памяти и на диске (json)

Функция обновления списка новостей проверяет дату и время последней модификации файла json.
Если отличия от текущей даты и времени расходятся на уровне не ниже часа, значит
будет вызвана функция read_news (непосредственно парсер сайта) для чтения новостей
с указанного ресурса. После чтения данные сохраняются в файл json.
Если дата и время с точностью до часа совпадают, проверяем что список новостей
в свойстве класса news_list не пуст, тогда возвращаем его. Если же пуст
(чтение еще не выполнялось), то читаем новости из файла.

После обновления данных функция get_news возвращает нужную новость из списка по ее индексу.

Если бот получает неверную команду или просто сообщение, выводится список команд
